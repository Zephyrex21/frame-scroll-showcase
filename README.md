## What Is This?

A premium animated brand page built for a **fictional luxury watch house вҖ” AURIENT GenГЁve**. As the user scrolls, 240 individual video frames render on an HTML5 Canvas in real-time, creating the illusion of the watch **exploding into its components and reassembling** вҖ” all driven by the scroll position.

Think Apple's iPhone product pages. Applied to a luxury watch brand.

> **This repo is a showdown.** It demonstrates that animated, immersive brand experiences can be built on the open web вҖ” no WebGL, no Three.js, no expensive animation tools. Just React, Canvas, and clean engineering.

## The Animation Engine

```
User scrolls в”Җв”Җв–ә scroll progress (0вҶ’1)
                      в”Ӯ
                      в–ј
            frameIndex = Math.floor(progress Г— 239)
                      в”Ӯ
                      в–ј
        requestAnimationFrame вҶ’ canvas.drawImage(frames[i])
                      в”Ӯ
                      в–ј
        7 text phases sync to frame ranges (DOM direct write, zero React re-renders)
```

240 JPEG frames are preloaded on mount (first 30 priority, rest parallel). Every scroll event is wrapped in `requestAnimationFrame` with a ticking guard вҖ” the canvas never redraws more than 60fps regardless of scroll speed.

## Features

### рҹҺ¬ Scroll-Driven Canvas Animation
- 240 frames rendered on `<canvas>` вҖ” cover-fit, always centred
- Scroll-progress mapped to frame index via `requestAnimationFrame`
- Sticky hero section: `height: calc(100vh + 5500px)` with `position: sticky` inner

### рҹ“– 7 Scroll-Synced Text Phases
Each phase fades in/out based on the current frame range вҖ” no timers, no GSAP:

| Phase | Frames | Content |
|-------|--------|---------|
| 0 | 0 вҖ“ 38 | Brand reveal вҖ” AURIENT wordmark |
| 1 | 32 вҖ“ 78 | "346 Components. One Movement." |

| 5 | 182 вҖ“ 222 | Founder's quote |

### рҹҢ“ Light / Dark Theme
- CSS custom properties on `[data-theme]` вҖ” zero JS overhead for theme switch
- Persisted in `localStorage`
- Warm parchment (`#F5F0E8`) vs void black (`#080604`)

### вңЁ UI Details
- **Custom cursor** вҖ” gold dot + elastic lagging ring (no CSS cursor, pure RAF loop)
- **Floating section nav** вҖ” dot indicators, hover labels, active state from IntersectionObserver
- **Animated counters** вҖ” 60 / 31 / 72h / 100 count up on scroll-in

## Quick Start

### Prerequisites
- Node.js вүҘ 18
- Python 3 + Pillow (`pip install Pillow`)
- Your source frame ZIP (the 240 PNG frames)
- MongoDB (optional вҖ” app runs fully without it)

### 1. Clone
```bash
git clone https://github.com/YOUR_USERNAME/aurient-animated-brand-showcase.git
cd aurient-animated-brand-showcase
```

### 2. Generate frames
```bash
python3 setup_frames.py /path/to/your/frames.zip
```
This converts the source PNGs вҶ’ progressive JPEGs (~15MB total) and places them in `backend/public/frames/`.

### 3. Install & run
```bash
npm install          # installs concurrently at root
npm run install:all  # installs backend + frontend dependencies

npm run dev          # starts both servers concurrently
```

| Service | URL |
|---------|-----|
| Frontend | http://localhost:5173 |
| Backend API | http://localhost:5000/api/watch |
| Frames | http://localhost:5000/frames/frame_00000.jpg |

### 4. (Optional) Seed MongoDB
```bash
cd backend && npm run seed
```

## Can I Build This For My Brand?

Yes. This architecture works for **any product** with a sequence of frames вҖ” a car, a sneaker, a perfume bottle, a phone. The stack is intentionally minimal so it can be adapted quickly.

What you bring:
- Your brand's video/animation exported as PNG frames (ideally 120вҖ“300 frames at 24fps)
- Your brand identity (colors, typography, copy)

What this repo provides:
- The entire scroll engine, text-phase system, and site structure
- Ready to clone, adapt, and deploy
